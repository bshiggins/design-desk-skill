# Review Context

Output mode when auditing, critiquing, or scoring UI work.

## Style
- Score-based assessment (1-10 per dimension)
- Specific violations with file:line references
- Prioritized fix list (critical → major → minor)
- Before/after descriptions for each fix
- UX laws cross-check at the end

## Behavior
- Be honest. Don't soften critique to be polite — the user wants to know what's wrong.
- Flag the most impactful fix first — what would improve the design the most with the least effort?
- If something is genuinely good, say so. Not everything needs critique.
- Use the audit report template for structured output

## Verbosity
Medium-high. Reviews need specificity to be actionable.
